문제 14

FROM students

문제 15
students 테이블에서 dept_id별 평균 점수를 구하되, 평균이 80 이상인 학과만 출력하라.

답 = SELECT dept_id, AVG(score) AS 평균점수
FROM students
GROUP BY dept_id
HAVING AVG(score) >= 80;

주관식 문제

주관식 1
SQL 명령어는 기능에 따라 DDL, DML, DCL로 분류된다. CREATE, SELECT, DROP, ALTER를 각각 해당 분류에 넣으시오.

DDL (Data Definition Language): CREATE, DROP, ALTER (테이블 등 구조 정의)

DML (Data Manipulation Language): SELECT (데이터 조작/조회)

참고: SELECT는 보통 DML에 포함시키지만, 학술적으로는 DQL(Data Query Language)로 따로 분류하기도 합니다.





주관식 2
PRIMARY KEY의 특성 두 가지를 쓰시오.

주관식 2: PRIMARY KEY(기본키)의 특성
Not Null (비어있음 방지): 기본키 값은 NULL이 될 수 없습니다.

Unique (유일성): 테이블 내의 모든 튜플(행)에 대해 중복된 값을 가질 수 없습니다.



주관식 3
아래 SQL에서 빈칸에 들어갈 키워드를 쓰시오.

주관식 3: SQL 빈칸 채우기
정답: HAVING

해설: GROUP BY로 그룹화된 결과에 대해 필터링(조건)을 걸 때는 WHERE가 아닌 HAVING 절을 사용합니다.




SELECT dept_id, AVG(score)
FROM students
GROUP BY dept_id
( ? ) AVG(score) >= 80;

주관식 4
트랜잭션의 ACID 특성 4가지를 쓰고, 각각을 한 문장으로 설명하시오.

Atomicity (원자성): 트랜잭션의 작업이 부분적으로 성공해서는 안 되며, 전부 수행(All)되거나 전혀 수행되지 않아야(Nothing) 합니다.

Consistency (일관성): 트랜잭션 완료 후에도 데이터베이스는 미리 정의된 규칙(제약 조건 등)에 따라 일관된 상태를 유지해야 합니다.

Isolation (격리성): 수행 중인 트랜잭션이 완료되기 전까지는 다른 트랜잭션이 그 결과를 참조할 수 없습니다.

Durability (지속성): 성공적으로 완료된 트랜잭션의 결과는 시스템 장애가 발생하더라도 영구적으로 기록되어야 합니다.





주관식 5
아래 SQL의 실행 결과로 COUNT(*)와 COUNT(score)에서 각각 어떤 값이 출력되는지 쓰고, 두 결과가 다른 이유를 설명하시오.

COUNT(*): 5 (NULL을 포함한 전체 행의 개수를 셉니다.)

COUNT(score): 3 (NULL 값을 제외하고 실제 데이터가 있는 행만 셉니다.)

이유: COUNT(*)는 레코드 자체의 개수를 세는 반면, 특정 컬럼명을 지정하면 해당 컬럼이 NULL인 경우는 카운트에서 제외하기 때문입니다.



-- score 컬럼 값: 85, NULL, 90, NULL, 75
SELECT COUNT(*), COUNT(score) FROM test_table;


주관식 6
FOREIGN KEY 제약조건의 역할을 설명하고, FOREIGN KEY 컬럼에 NULL이 허용되는지 여부와 그 이유를 쓰시오.

역할: 두 테이블 간의 관계를 맺어주고, 참조 무결성을 유지합니다. (참조하는 테이블에 실제 존재하는 값만 입력되도록 제한)

NULL 허용 여부: 허용됩니다.

이유: 외래키가 반드시 특정 행을 참조해야 한다는 제약이 없다면(예: 아직 소속 학과가 결정되지 않은 신입생), NULL을 통해 '참조 대상이 없음'을 나타낼 수 있습니다.





주관식 7
아래 SQL이 내부적으로 처리되는 실행 순서(FROM, WHERE, GROUP BY, HAVING, SELECT, ORDER BY)를 올바르게 나열하시오.


정답: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

해설: 데이터를 먼저 가져오고(FROM), 개별 행을 필터링한 뒤(WHERE), 그룹화하여(GROUP BY) 그룹에 조건을 걸고(HAVING), 보여줄 컬럼을 결정하고(SELECT), 마지막으로 정렬(ORDER BY)합니다.




SELECT dept_id, AVG(score)
FROM students
WHERE grade >= 2
GROUP BY dept_id
HAVING AVG(score) > 80
ORDER BY AVG(score) DESC;

주관식 8
아래 설명이 가리키는 데이터베이스 용어를 쓰시오.

릴레이션에서 튜플을 유일하게 식별할 수 있는 속성들의 집합으로, 유일성과 최소성을 모두 만족해야 한다.

정답: 기본키 (Primary Key) 또는 후보키 (Candidate Key)

추가 설명: 문제에서 유일성과 최소성을 모두 만족해야 한다고 했으므로 가장 정확한 용어는 후보키이며, 그중 선택된 것이 기본키입니다.




주관식 9
WHERE와 HAVING의 차이점을 GROUP BY 실행 순서와 연관지어 설명하시오.

WHERE: GROUP BY가 실행되기 전에 개별 레코드들에 대해 필터링을 수행합니다. 집계 함수를 사용할 수 없습니다.

HAVING: GROUP BY가 실행된 후, 생성된 그룹들에 대해 필터링을 수행합니다. 집계 함수(AVG, SUM 등)를 사용한 조건 설정이 가능합니다.




주관식 10
GROUP BY 절을 사용할 때 SELECT 절에 올 수 있는 항목과 올 수 없는 항목을 구분하여 설명하시오.

올 수 있는 항목: GROUP BY 절에 사용한 기준 컬럼, 그리고 SUM(), AVG(), COUNT() 등과 같은 집계 함수.

올 수 없는 항목: 그룹화 기준이 아닌 개별 컬럼(예: 학과별로 묶었는데 학생 개개인의 이름을 SELECT에 적을 수 없음). 이는 그룹 내 여러 데이터 중 어떤 것을 보여줘야 할지 결정할 수 없기 때문입니다.
